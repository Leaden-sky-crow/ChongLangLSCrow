# 功能规格说明

## 1. 管理后台站内通知功能

### 1.1 功能概述
在管理后台添加完整的站内通知发送功能，允许管理员向指定用户或全体用户发送系统通知。

### 1.2 功能需求
- **通知类型支持**：
  - 全体用户通知（广播）
  - 指定用户通知（通过用户ID或用户名搜索）
  
- **通知内容**：
  - 标题（必填，最大50字符）
  - 内容（必填，最大500字符）
  - 类型：`system`（系统通知）
  - 可选链接URL（点击通知后跳转）

- **用户选择**：
  - 单选模式：选择特定用户
  - 多选模式：选择多个用户
  - 全选模式：发送给所有用户

- **UI组件**：
  - 用户搜索输入框（支持按用户名、邮箱搜索）
  - 用户列表显示（带头像、用户名）
  - 通知表单（标题、内容、链接URL输入）
  - 发送按钮和取消按钮
  - 发送成功/失败反馈

### 1.3 技术实现
- **前端**：
  - 创建 `NotificationsManagement` 组件
  - 使用 `Select` 组件支持多选用户
  - 表单验证（标题和内容必填）
  - 调用 server action 发送通知
  
- **后端**：
  - 扩展现有的 `createNotification` server action
  - 添加批量发送功能 `sendSystemNotifications`
  - 添加用户搜索 API `searchUsersForNotifications`

- **数据库**：
  - 利用现有的 `notifications` 表
  - RLS 策略已支持管理员插入

### 1.4 用户体验
- 表单提交后显示 loading 状态
- 发送成功后清空表单并显示成功提示
- 发送失败时显示具体错误信息
- 支持取消操作

## 2. 主页分类切换自动滚动功能

### 2.1 功能概述
当用户在主页点击分类按钮时，自动平滑滚动到分类内容区域，避免手动下滑的操作。

### 2.2 功能需求
- **触发条件**：在主页 (`/`) 点击分类筛选按钮
- **目标位置**：滚动到 `#content` 容器（包含 CategoryFilter 和 PostList 的区域）
- **滚动效果**：平滑滚动，持续时间 500ms
- **兼容性**：支持所有现代浏览器

### 2.3 技术实现
- **前端**：
  - 修改 `CategoryFilter` 组件
  - 在路由跳转后添加滚动逻辑
  - 使用 `window.scrollTo` 或 `element.scrollIntoView` 实现平滑滚动
  - 添加防抖机制避免重复滚动

- **滚动策略**：
  - 方案A：路由跳转后延迟滚动（推荐）
  - 方案B：使用 Intersection Observer 监听内容加载完成
  - 选择方案A，简单可靠

### 2.4 用户体验
- 滚动过程平滑自然
- 不影响其他页面的正常行为
- 在 `/posts` 页面保持原有行为（不自动滚动）
- 移动端和桌面端都适用
